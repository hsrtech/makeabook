What’s new changed in Bootstrap 4?

Lets take a quick look at whats added, or changed in Bootstrap 4, so far. There may be some more changes by the time we get final release of Bootstrap 4, but most of them are already being implemented.

Moved from Less to Sass

Bootstrap 4 is using Sass as its CSS pre-processor, instead of Less used in previous versions. Sass is faster in compilation, and have a bigger community then Less.

Improved Grid System

Grid system is improved to provide better support for mobile devices.

Opt-In Flexbox support

Bootstrap now allow you-to use flexbox, just by changing a variables value in Sass file. Flexbox is now fully supported by all modern web browsers. In Internet Explorer, its supported from version 10 onwards.

Dropped wells, thumbnails, and panels for cards

A new component “Cards”, has replaced old components: wells, thumbnails, and panels.

Introduced Reboot

In previous version Bootstrap was using Normalize.css, they have now extended normalize.css and created their own version called reboot.css, reboot is designed specially to meet requirements of bootstrap users.

Customization options

Bootstrap 4 allow lots of customization with help of variables in Sass. For example you can change value of rounded corners everywhere, by just changing one variable’s value in Sass, and recompile it.

Dropped IE8 Support

Bootstrap 4 had dropped support for Internet Explorer 8, because of its lack for CSS3 features.

Moved to rem and em units

Pixels have been swapped for rems and ems where appropriate, to make responsive typography and component sizing easier.

All JavaScript plugins are rewritten

All JavaScript plugins in bootstrap 4 are rewritten using latest EcmaScript 6 standards, with lots of improvements.

Improved auto positioning of tooltips, and popovers

Bootstrap 4 is utilizing another JS library called “Tether” for improving auto positioning of tooltips, and popovers.

Dropped Glyphicons

Bootstrap has dropped support for Glyphicons, because most people are using font awesome instead of glyphicons. A discussion is going on to add or not to add support for other icon fonts in upcoming versions.

Improved styling of form elements

WTF Forms, are now integrated directly inside Bootstrap 4, making forms more beautiful, and cross browser consistent.

Modular Architecture / Components

Bootstrap 4 has improved its architecture a lot, they have adopted a modular approach. Reusable elements are now modularized, and named components. It will be a lot easier to create your own components using Bootstrap 4.

New Utility Classes

New utility classes are added. They’re designed to reduce the frequency of highly repetitive declarations in your CSS.
